日期及时间的格式化

时间选择组件的显示样式及功能是根据 displayFormat 中的关键字的不同而不同，通过组合 format、displayFormat 属性实现个性化的时间格式及功能。

格式化关键字有：yyyy、M、MM、d、dd、h、hh、m、mm、s、ss。它们分别代表年、月、日、时、分、秒，时间选择组件会分别去匹配这些关键字，来实现选择结果的格式化。 因此，你可以传入关键字任意组合，个性化定制选择结果，如：
😇hh😍mm😳ss😜yyyy😂MM😎dd😏 也是合法的。

注意：
  1. 当 displayFormat 为空时，以 format 的值为准。即当 format 和 displayFormat 一样时，只传入 format 属性即可。
  2. 传入时间选择组件的 value 必须和 displayFormat 的格式保持一致，才能正确解析出传入的时间，但有一些特殊性况，是不能正确解析的。如：

    displayFormat 为 yyyyMd，value 为 201913，这样的情况，会识别为 2019 年 1 月 3 日。但当 value 为 201912 时，正确的结果应该是 2019 年 1 月 2 日，但程序比较难枚举所有意外，所以组件采用贪婪匹配的方式，会识别为 2019 年 12 月，并且匹配不到日期。

    如何避免出现上述情况呢？一般会采用分割符，如传入 displayFormat 为 yyyy/M/D，则 value 为 2019/1/2。因为时间选择组件不会匹配非关键字以外的字符，所以即可识别出正确的时间。
