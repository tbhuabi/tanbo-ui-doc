名词解释

为了在使用中更好的利用 TBus 提供的一系列能力，准确的理解 TBus 中和各种概念是非常必要的。下面我们分别对基础数据、交互能力、选区、渲染及其它相关的名词术语作一一解释。

基础数据

一段固定的数据结构样板，并且自已保存需要的数据，并实现根据自身的数据渲染对应虚拟 DOM 树。
模板必须从属于一个 Fragment。同时，模板也可以有任意个自己的子插槽，即 Slot，模板根节点与 Slot 之间的固定结构不可编辑，但每一个 Slot 都是一个可编辑的区域。
Fragment 为一段可编辑区域，可以插入任意的模板和字符串。同时，Fragment 还保存了对应内容的 FormatMap。
插槽为一个抽象的概念。在 TBus 中，一个插槽，意味着一段可编辑的区域（即 Fragment），它们总是呈一对一映射关系的。
如一个无序列表 ul，每一个子插槽都是一个 li，li 的内容可以是变化的，但 ul 的子级总是 li，这是不可变的。
模板转换器负责把一段匹配到的 HTML 元素，转换为对应的 Template 实例。当我们要扩展 TBus 的结构样板时，可以自定义一个转换器，并添加在 TBus 配置项的 templateTranslators 中。
TBus 所有的核心模板也都是以同样方式实现的。
格式转换器负责把匹配到的格式转换为抽象的格式数据（FormatAbstractData），并根据不同的上下文及状态渲染相应格式。当我们要扩展格式的时候，就可以通过继承 BlockFormatter 或 InlineFormatter，并把对应实例添加到 TBus 配置项的 Formatters 选项中，来实现自己想要的格式。
标记一段格式的范围及状态

交互能力

用于操作 TBus 的工具类，当用户点击或选择工具类时，执行对应的 Command 命令。
提供了操作 TBus 抽象数据的能力。当用户操作某项工具时，用于响应并执行对应操作。
用于查询选区与工具匹配规则之间的状态，分为三种：Disabled、Highlight、Normal
当用户触发或 TBus 运行到某些关键节点时，所广播的事件，监听这些事件，并作出操作，可扩展出除了 Commander 以外额外的能力。
当用户输入、删除、粘贴等一系列可能修当前数据的行为时，会在当前虚拟 DOM 节点上触发相应事件，并逐级向上冒泡，直到根节点，或取消冒泡。

选区

用于记录一段或多段已选择的范围（TBRange），并提供了一系列针对选区的操作。
用于选择一段数据，并执行如：删除、查找范围、合并等操作。

渲染

用于渲染一段抽象数据（Fragment）。将 Fragment 生成虚拟 DOM，并对新旧两次的虚拟 DOM 树作 diff 运算，求出最小更新范围，并更新真实 DOM。
Renderer 对象还提供了一系列的查询方法，这些方法可以方便的获取 Fragment、VDom、Template 与真实 DOM 节点之前的关系。
